第一次部署项目

2026年5月15号是我跟进了一年的项目的交付日期。之前在学校里，无论是课设还是大作业。都是在自己电脑上跑通并得到一个不错的结果，老师就会很满意。因此之前我完全没有接触过实际迁移部署方面的工作，因此第一次做废了不少力气。

一、项目简介

我们做的项目主要是某省的新能源发电功率预测，该项目主要有两部分构成:

  • 前后端+数据库：这部分主要是由开发组完成的，然后他们发给我们一个docker镜像，我们部署在自己的服务器上。经过端口映射后，我们可以通过3306端口访问数据库并通过80端口查看前端页面。
  • 定时调度预测：这个部分统一放在了一个固定的文件夹下， 每天会定时调度预测函数去获取预测所需的数据、加载模型、得到预测结果，最后写入数据库。

二、部署过程

之前从来没有接触过使用docker打包一个项目，只接触过使用docker部署项目。因此这是我第一次接触工程部署的过程（注意，我们所用的所有服务器都是离线的）。

四、部署心得

这次项目交付让我第一次真正接触到工程部署和项目迁移。以前在学校做课设或实验时，更多关注的是代码能不能跑通、模型指标是否好看、结果是否符合预期。但真实项目部署需要考虑的问题更多，包括：

  1. 代码是否完整；
  2. Python 环境是否一致；
  3. 依赖包是否能安装；
  4. Docker 镜像是否能离线构建；
  5. 数据库是否完整备份；
  6. CSV、模型文件、配置文件是否遗漏；
  7. 目标服务器的端口是否正确映射；
  8. 容器启动后日志是否正常；
  9. 程序能否持续运行并定时写入数据库。

这次我最大的收获是理解了 Docker 在项目迁移中的作用。Docker 并不只是“启动别人给的镜像”，更重要的是可以把项目运行环境、代码和启动命令封装成一个整体，从而减少不同服务器之间环境不一致带来的问题。

通过这次部署，我也意识到离线部署一定要提前准备好所有依赖，包括 Docker 基础镜像、Python 环境、项目文件、数据库备份和外部数据文件。只要其中一个环节遗漏，到了目标服务器上就可能因为无法联网而无法补救。

因此，对于离线项目交付，最重要的原则是：不要假设目标服务器可以临时下载任何东西，而是要把项目运行所需的一切都提前打包好。
